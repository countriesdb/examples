Prefer Local Variant - CommonJS

This example displays local variant names for official subdivisions when available by setting preferLocalVariant: true in window.CountriesDBConfig. Local variants are alternative names that may be more familiar to users (e.g., "Cairo" instead of "Al Q훮hirah"). Note: This only works with official ISO subdivisions, so we use data-prefer-official on the subdivision select. This option is ignored when extended subdivisions are active.

Try it: Select Egypt (EG) to see local variants. For EG-C, you'll see "Cairo" (local variant) instead of "Al Q훮hirah" (official name). Similarly, EG-ALX shows "Alexandria" instead of "Al Iskandar카yah", and EG-GZ shows "Giza" instead of "Al J카zah".

This option works alongside romanization preference: romanization system is selected first, then local variant is preferred if available for that system.
